details

What it is: Claude Code now supports "routines" — prompts that run on Anthropic's web infrastructure without your laptop open, triggered by schedules, API calls, or GitHub events.

How to set them up:

  • Create from three places: Claude Code terminal (`scheduled triggers`), the web at claude.ai/code, or the desktop app
  • Desktop app → Scheduled Tasks → New Task → choose Local or Remote
  • Configure: name, prompt (the actual instruction), model, repository, and cloud environment
  • Set cadence: hourly, daily, weekdays (minimum interval is once per hour)
  • Connect integrations: Slack, Gmail, or custom API endpoints with your API keys
  • Set permissions for how Claude should act autonomously

Key gotchas Nate discovered:

  • These are one-shot prompts — Claude can't stop and ask you questions, so your prompt must be self-contained
  • Existing local automations don't just migrate over and work — you'll need to adapt them
  • Remote tasks run inside a GitHub repository context; local tasks run on your machine's schedule
  • Nate showed 4 local tasks and 4 remote (GitHub-based) routines running simultaneously

What Nate accomplished:

  • Migrated his existing automations to run entirely on Anthropic's infrastructure
  • Set up both local and remote routines from the desktop app
  • Demonstrated the full configuration flow including connector setup

Tools & links:

  • Nate's AI Automation Society (Free) — free resources and community
  • Nate's AI Automation Society Plus — full courses and support
  • Uppit AI — Nate's agency, work-with-me option
  • Claude Code Routines — available at claude.ai/code or via desktop app scheduled tasks

Why it matters for you: This is directly relevant to your mx-workflow setup — you could run `/mx:*` commands as headless routines on a schedule without keeping a terminal open, and trigger builds or validations from GitHub events.

details

What it is: Selling advertising space on shared postcards mailed to local homeowners via USPS. A 25-year-old guest turned $90 into $50K/year doing this part-time.

How it works:

  • Design a shared postcard (front and back) with ad slots for local businesses
  • One business per category — no competing advertisers on the same card (e.g., only one landscaper)
  • Each business gets full exposure for their services on the card
  • Mail postcards using USPS Every Door Direct Mail (EDDM) — no mailing list or names needed, just target a zip code/neighborhood
  • Example: one card went to 10,000 homeowners in a nearby city

How he finds clients:

  • Uses a "weird Facebook trick" to make local business owners DM him first (not detailed fully, but involves positioning in local Facebook groups)
  • Closes 95% of deals over Facebook Messenger or text — no phone calls, no Zoom
  • One plumber paid $1,100 after just 5 text messages

Economics:

  • Started for under $100 (under $90 specifically)
  • Pockets roughly 50% profit on everything collected
  • No LLC needed at first — he sent his first two cards without one
  • No website, no logo, no fancy branding — just proved the model makes money first

Key insight from Chris: This is like a newsletter but physical — a two-sided marketplace where you need supply (the homeowner eyeballs via USPS) and demand (the local businesses paying for ads). Chris compared it to how his own multi-million dollar business was built with just 5-6 pieces of direct mail to a few thousand repair shops.

Tools & links:

  • USPS Every Door Direct Mail — free USPS tool to target mail routes by zip code without a mailing list
  • beehiiv — newsletter platform Chris uses (sponsor, 30% off first 30 days)
  • Chris's newsletter — The Koerner Office Podcast newsletter

Why it matters for you: Zero-tech barrier to entry. You could test this in your local area for under $100 this weekend — design a postcard, find 4-6 local businesses on Facebook, and mail via EDDM. The margins are real and it doesn't compete with your engineering time.
