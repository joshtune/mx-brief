Planning In Claude Code Just Got a Huge Upgrade

54.4K +20.4K
details

Ultra Plan: cloud-based planning crushes local

Ultra Plan is a new Claude Code feature that offloads planning to the web. Flow: start plan from CLI → it opens in Claude Code on the web → draft/review with emoji reactions and inline comments → approve → plan teleports back to your terminal for local execution. In Nate's test, Ultra Plan finished in ~1 minute vs. 4+ minutes for local planning on the same prompt. The web-planned version was also faster at execution because the plan was clearer and more structured. The web interface shows context, existing code analysis, new files to create, modifications, verification steps, and sometimes diagrams. It syncs to your Git repo so it can explore your directory structure remotely. Key insight: Ultra Plan reads your codebase via Git sync, finds existing skills/scripts, and leverages them in the plan. For mx-workflow: this could replace or complement /mx:plan — instead of local planning, you could integrate Ultra Plan for complex multi-file features.

The Next Evolution of AI Coding Is Harnesses - Here's How to Build Them

13.2K
details

Archon: open-source harness builder for AI coding

Cole completely overhauled Archon (formerly an 'AI command center') into the first open-source harness builder for AI coding. A harness is the orchestration layer on top of coding agents that makes AI coding deterministic and repeatable. Archon workflows are DAGs of nodes — each node is either a prompt sent to a coding agent session or a deterministic command (context curation, validation, test running). Example workflow: plan → implement → test loop → review → human approval gate → PR creation. Archon ships with prepackaged workflows: fix GitHub issues, create PRs from ideas, PR validation/review, full PRD creation with human-in-the-loop. Custom workflows are simple to create. It includes a Claude Code skill so you just say 'use archon to build this feature' and it auto-selects the right workflow. Repo linked in description. For mx-workflow specifically: Archon's workflow node model (prompt nodes + deterministic command nodes) is directly comparable to your slash command architecture. The human approval gate pattern, the plan→implement→test loop, and the PR creation flow could all inform mx-workflow improvements. Study Archon's node composition model — it might be cleaner than sequential slash commands for complex multi-step workflows.
