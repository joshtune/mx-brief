Ollama + Claude Code = 99% CHEAPER

112.2K +26.6K
details

Run Claude Code free with local models or OpenRouter

What it is: Two methods to run Claude Code without paying for Anthropic's API — using local open-source models via Ollama, or using free models through OpenRouter.

The car analogy (Nate's explanation):

  • Claude Code = the car (harness, tools, file organization, planning)
  • The AI model = the engine (Opus, Sonnet, Haiku by default)
  • You can swap the engine while keeping the car
  • Token limits and subscription costs come from the engine, not the car

Method 1: Ollama (local models)

  • Download open-source models to run entirely on your machine
  • No internet required, no token limits (hardware is the limit)
  • Open-source = you can see, edit, modify, run the model
  • Free because compute happens on your hardware

Method 2: OpenRouter (cloud, free tier)

  • Route through OpenRouter's API to access free cloud-hosted models
  • Still runs through Claude Code's harness
  • Limits work differently than Anthropic's direct API

Key distinction:

  • Open-source models (downloadable, inspectable): used with Ollama
  • Closed-source models (Sonnet, GPT, Gemini): API-only, pay per token
  • You lose quality going from Opus → local models, but gain unlimited usage

Tools & links:

  • Ollama — local model runner
  • OpenRouter — multi-model API gateway with free tiers

Why it matters for you: If you're hitting Claude Code token limits or want unlimited local iteration for non-critical tasks, Ollama is the play. Use it for drafting and iteration, keep Opus for final-pass quality work.

What a harness is:

  • The system around the coding agent — not the agent itself
  • Turns "manually shepherding an agent through 8 steps" into "run one command"
  • Makes AI coding deterministic and repeatable across codebases
  • Combination of prompt nodes (sent to agent) and deterministic nodes (enforced commands)

How Archon works:

  • Every workflow = a combination of nodes
  • Node types: prompt (sent to coding agent session) or deterministic command (forced execution)
  • Example workflow: plan → implement → test loop → review → human approval gate → PR
  • Human approval gates can be built into workflows
  • Comes with a Claude Code skill — say "use archon to build this feature" and it auto-invokes the right workflow
  • Logs available for monitoring workflows in progress

Pre-packaged workflows included:

  • Fix GitHub issues
  • Create PRs from ideas
  • PR validation and review
  • Full PRD creation with human-in-the-loop
  • Custom workflow creation

Architecture:

  • Workflows run across codebases
  • Parallel task handling supported
  • Archon handles session orchestration behind the scenes
  • Can encode any development process as a workflow

Tools & links:

  • Archon (linked in video description) — open-source harness builder
  • Claude Code — the underlying coding agent

Why it matters for mx-workflow: This is directly relevant to your project. Archon's node-based workflow model (prompt nodes + deterministic nodes + human gates) is a more structured version of what mx-workflow does with slash commands and agents. Key differences to evaluate:

  • Archon has explicit deterministic nodes for validation (you don't leave linting/testing to the agent's memory)
  • Parallel task handling is built in
