3 MacBooks Did What One Never Could

658

Clustering three M5 Max MacBook Pros for local LLM inference.

Direct hardware answer: three M5 Max MacBook Pros (128GB each) clustered over Thunderbolt 5 beat a single machine on 122B param models — relevant cost comparison to Mac Studio/Mac mini setups.

details

What it is: Alex wires three M5 Max MacBook Pros (128GB RAM each) into a distributed inference cluster using Thunderbolt 5 + RDMA, running models up to 122B params across them. Framed as a preview of what an M5 Max Mac Studio might deliver before WWDC 2026.

Hardware setup:

  • 3× M5 Max MacBook Pro, 128GB unified memory each (384GB total pool)
  • 3× Thunderbolt 5 cables in a mesh topology
  • RDMA enabled for low-latency inter-node transfer
  • Jackal host file defines the node layout for MLX distributed

How it works:

  • MLX = Apple Silicon's ML framework; supports distributed inference across nodes
  • Environment must match exactly across all machines (Alex had Claude replicate it from his docs)
  • Tensor parallelism shards model layers across machines — pays overhead per layer, so tiny models see less benefit

Benchmarks:

  • Qwen 34B-4B (2GB): 179 tok/s on 1 node → 220 tok/s on 2 nodes (+22%)
  • Qwen 3.5 122B-A10B MoE (65GB on disk): 57 tok/s on 1 node → 73 tok/s on 2 nodes (+27%)
  • MoE models don't scale linearly because active expert size is smaller than total params
  • Transcript cuts off before the full 3-node numbers and the M5 Max vs M3 Ultra cluster head-to-head

Tools & links:

  • MLX — Apple's ML framework for Silicon
  • Alex's GitHub has step-by-step cluster setup docs (referenced but URL not in transcript snippet)

Cost context for your focus: M5 Max MacBook Pro 128GB starts around $4,500-5,000; three of them = ~$14-15K for 384GB pooled VRAM. A Mac Studio M3 Ultra 512GB is ~$10K single-box. A Mac mini M4 Pro maxes at 64GB (~$2K) — not in the same league for 100B+ models. The cluster buys you headroom a mini cannot touch, but at 7× the price of a loaded mini.

Why it matters for you: If you're sizing a local AI box vs. a Mac mini, this is the upper bound — clustering high-end MacBook Pros gets you 122B-param inference at usable speeds, but the price delta vs. a Mac mini (or even a single Mac Studio) is the real story.

Why Japan is Giving Free Houses

812

Japan giving away 8M+ abandoned houses called akiya.

Niche real estate arbitrage opportunity — free/cheap Japanese homes nobody wants, ripe for a flip-and-rent or remote-work play.

details

What it is: Japan has 8M+ abandoned homes (called *akiya*, 空き家) that the government and rural towns are giving away free or selling dirt cheap.

How it works:

  • Owners aged out and passed away; heirs inherit but don't want the property
  • Status/social stigma: heirs who climbed social classes refuse to claim homes in poor or rural neighborhoods (don't want to be associated)
  • Result: massive surplus of vacant housing stock that municipalities are unloading
  • Government and local programs list these homes via *akiya banks* — sometimes free, sometimes nominal price, often with renovation grants attached

The arbitrage angle (not in transcript, but the obvious play):

  • Buy/claim akiya → renovate → rent to remote workers, tourists, or list on Airbnb
  • Some foreigners have built content brands documenting akiya renovations (huge YouTube niche)
  • Yen weakness + Japan tourism boom = favorable macro

Tools & links:

  • Akiya banks (空き家バンク) — municipal listing sites, search by prefecture
  • Akiya & Inaka — English-language akiya marketplace
  • Episode itself was light on transcript — host barely covered the mechanics, mostly vibe-checking the concept

Why it matters for you: Asymmetric bet — low capital, high optionality real estate play in a developed country. Engineering brain + remote income = you could actually pull this off.

Breaking changes:

  • None user-facing, but note the native binary distribution — if you pin/vendor the JS bundle, your install path changes

  • `Bash(find:*)` allow rules no longer auto-approve `find -exec` / `-delete`
