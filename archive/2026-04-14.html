v2.1.107

Plugin monitors, PreCompact hooks, worktree improvements, bug fixes.

Background monitors and PreCompact hooks unlock new plugin automation patterns for your mx-workflow commands.

details

What changed:

  • Background monitor support for plugins via top-level `monitors` manifest key — auto-arms at session start or on skill invoke
  • `PreCompact` hook support: hooks can block compaction by exiting with code 2 or returning `{"decision":"block"}`
  • `EnterWorktree` tool now accepts a `path` parameter to switch into an existing worktree
  • `/proactive` is now an alias for `/loop`
  • `WebFetch` strips `<style>` and `<script>` contents so CSS-heavy pages no longer exhaust the content budget
  • Stalled API streams abort after 5 min of no data and retry non-streaming instead of hanging
  • `/doctor` layout improved with status icons; press `f` to have Claude fix issues
  • Skill description listing cap raised from 250 to 1,536 characters
  • Stale agent worktrees from squash-merged PRs now get cleaned up
  • Thinking hints shown sooner during long operations (v2.1.107)

Bug fixes:

  • Images attached to queued messages no longer dropped
  • Screen no longer goes blank when prompt input wraps to a second line
  • Leading whitespace preserved in assistant messages (fixes ASCII art, indented diagrams)
  • alt+enter and Ctrl+J newline insertion fixed (regression from 2.1.100)
  • One-shot scheduled tasks no longer re-fire repeatedly
  • Marketplace plugins with `package.json` now auto-install dependencies after install/update
  • Marketplace auto-update no longer breaks when a plugin process holds files open

Breaking changes:

  • None

Links:

  • v2.1.107 release notes
  • v2.1.105 release notes

Why it matters for you: The new plugin `monitors` manifest key and `PreCompact` hooks give mx-workflow more control over session lifecycle — worth exploring for your slash commands and automation agents.
