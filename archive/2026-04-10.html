After This, 16GB Feels Different

234.3K

TurboQuant makes 16GB Macs run real models

Watch if you've been telling yourself your 16GB Mac Mini can't do local AI — TurboQuant changes that math.

details

What it is: Alex tests TurboQuant, a new compression technique that shrinks LLMs enough to run meaningfully on 16GB hardware.

The problem TurboQuant solves:

  • Qwen 3.5 9B (a top-performing open model) is 19.3GB uncompressed at BF16 — won't fit on a 16GB Mac Mini
  • Standard quantization helps: 8-bit drops it to 10GB, 4-bit to ~6GB
  • But even a 6GB model balloons in memory when you add KV cache for context — Alex showed a 4-bit model jumping from ~6GB to way more at full context length
  • Below 4-bit quantization, models start looping and spitting garbage

How TurboQuant is different:

  • Goes beyond traditional weight quantization
  • Handles compression more like image compression — visually (functionally) identical at a fraction of the size
  • Specifically targets the KV cache memory problem, not just model weights

Hardware tested:

  • 16GB Mac Mini — the constraint machine
  • 128GB Mac — his daily driver for comparison
  • 512GB machines — the big boys

Cost comparison to Mac Mini:

  • Mac Mini 16GB: ~$599-799 (entry point, now viable for small models with TurboQuant)
  • Mac Mini with more RAM: price jumps significantly per Apple's unified memory pricing
  • TurboQuant's pitch is you DON'T need to upgrade — squeeze more from what you have

Models mentioned:

  • Qwen 3.5 family (massive downloads, strong performance)
  • Various Bowski quantizations on Hugging Face

Why it matters for you: If you're running a Mac Mini for dev work and want local AI without dropping $2K+ on RAM upgrades, TurboQuant is the first compression approach that makes 16GB genuinely usable beyond toy demos.

Intel B70: 32GB VRAM for under $1K

details

What it is: Alex benchmarks Intel's new Arc Pro B70 GPU against Nvidia and AMD for local AI workloads.

The GPU lineup tested:

  • Intel Arc Pro B70 — 32GB VRAM, under $1,000. Needs external power. 68 GB/s memory bandwidth (lowest)
  • Intel Arc Pro B50 — smaller sibling, no external power needed, powers from PCIe bus alone
  • Nvidia RTX Pro 4000 Black — 24GB GDDR7, 672 GB/s bandwidth, $1,699 at MicroCenter (retail $1,999)
  • AMD Radeon AI R9700 — 32GB GDDR6, 640 GB/s bandwidth, ~$1,300
  • Nvidia RTX 5090 — 32GB, ~$4,000 (the comparison benchmark)

The killer math:

  • 4x Intel Arc Pro B70 = 128GB total VRAM for ~$4,000
  • 1x Nvidia RTX 5090 = 32GB VRAM for ~$4,000
  • That's 4x the memory for the same price
  • Tradeoff: Intel's memory bandwidth is significantly lower (68 GB/s vs 672 GB/s on Nvidia)

vs Mac Mini/Mac Studio:

  • Mac Mini M4 Pro (24GB unified): ~$1,599 — unified memory is simpler but non-upgradeable
  • Mac Studio M4 Ultra (192GB unified): ~$4,000+ — more memory but you're locked into Apple's ecosystem
  • Intel B70 route: upgradeable, standard PCIe, but you need a desktop that can power 4 GPUs
  • For pure VRAM-per-dollar on local LLMs, Intel wins — but Apple's unified memory has bandwidth advantages

Tools mentioned:

  • ChatLLM Teams — multi-model dashboard (sponsor)
  • Ollama (implied for local model serving)

Why it matters for you: If you're deciding between a maxed-out Mac or a GPU rig for local AI, Intel just made the GPU path 4x cheaper on VRAM. The bandwidth tradeoff matters for inference speed, but for running big models that just need to fit in memory, this is a game-changer.
