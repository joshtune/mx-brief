What it is: Live stream where Cole workshops a workflow that ingests one URL (HN story, article, product page) and produces a fully rendered video — voiceover, music, SFX, scene plans, and code.

How it works:

  • Claude Code writes the Remotion (React-based programmatic video) code per scene
  • Archon orchestrates the multi-agent pipeline and task hand-offs
  • Final render is a complete video file from a single prompt

Tools & links:

  • Remotion — React framework for programmatic video rendering
  • Archon — Cole's open-source agent orchestration platform powering the multi-stage workflow

What's different here:

  • Treats video generation as a *pipeline of specialized agents* rather than one mega-prompt
  • Uses Archon as the conductor — task graph + agent specialization instead of ad-hoc subagent calls
  • Claude Code is the codegen worker, not the orchestrator — clean separation of planning vs. execution
  • Multi-modal asset generation (audio, music, SFX, code) coordinated under one harness

Why it matters for you: mx-workflow could borrow Archon's orchestration-as-task-graph model — instead of one Claude session doing everything, split into planner → specialist agents → integrator, with explicit hand-offs you can inspect and rerun.

Earned-equity framework instead of giving away shares.

If your side hustle is graduating into a real business with hires, this is how to attract operators without bleeding your cap table.

details

What it is: The "earned equity" comp framework 8-figure founders use to hire execs without handing out free ownership.

How it works:

  • Don't say "here's 5%" — say "here's a path to 5% if you create measurable value"
  • Example COO package: $180K base + $120K target bonus + 5% equity earnout pool
  • Equity vests only on hard targets (e.g. EBITDA grows $2M → $4M, margins improve 5 points)
  • Pitch the upside in dollars: "this is worth $3–5M at exit" — makes the carrot concrete
  • If they miss, they still got paid well, but you keep your company

The principle:

  • "Incentives drive behavior" — Munger
  • Tie equity to outcomes you actually want, not just showing up

Tools & links:

  • Growth Accelerator Workshop (Contrarian Thinking) — pitched at the end for live comp-plan help; no URL given in transcript

Why it matters for you: The moment your side hustle needs a first real operator, you'll be tempted to overpay in equity because cash is tight — this framework lets you hire above your weight class without permanently diluting yourself.

Experimental live queries and query batching in remote functions.

Live queries could replace ad-hoc Supabase realtime wiring in your static-Pages-plus-Supabase setup.

details

What changed:

  • New experimental `query.live` function for live/reactive remote queries
  • `query.batch` now supported in `requested(...)`
  • Several fixes to remote form field defaults (`field.as('text', defaultValue)`, checkbox defaults)
  • Empty `.updates()` call now explicitly means "don't update anything"
  • `RemoteCommand` output type now unwraps `Promise`

Breaking changes:

  • On the server, the promise returned from `refresh` now represents adding the refresh to the map, not the time it takes to run the remote function — anything awaiting `refresh` for completion timing will resolve earlier than before

Links:

  • Release notes
  • PR #15705 (live queries + refresh change)
  • PR #15751 (query.batch)

Why it matters for you: If you lean on SvelteKit + Supabase, `query.live` is the kind of primitive that can simplify realtime UIs without hand-rolling subscriptions — worth prototyping early.
