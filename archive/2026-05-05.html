details

What it is: A live build walkthrough where Nate uses Claude Code to orchestrate ElevenLabs and ship a website-embedded voice agent trained on his 400-video YouTube transcript library — built in ~15 minutes from a one-line prompt.

The demo result:

  • Voice agent embedded on his website with a "Start Call" button
  • Trained on every transcript from his channel (~400 videos)
  • Answers questions conversationally about tools he's covered (e.g. recommended Firecrawl as a scraping tool, suggested a job-listings-to-Excel workflow as a beginner Claude Code project)
  • Claude Code handled: pulling transcripts, connecting to ElevenLabs API, configuring the agent, building required tools, embedding on the site

How a voice agent actually works (Nate's mental model):

  • It's a loop, not magic
  • Visitor speaks → mic captures audio → STT transcribes to text
  • LLM reads the text, decides to respond directly OR hit a tool/DB/vector store
  • Response goes back through TTS → user hears it → loop repeats

The 4 core pieces of every voice agent:

  • Persona — the system prompt (tone, role, personality, behavior rules)
  • Voice — pick from ElevenLabs voice library (or clone)
  • Knowledge base — drop in docs, OR wire to a vector store (Supabase, Pinecone, NotebookLM)
  • Tools — function calls the agent can invoke mid-conversation

Why Claude Code changes the game here:

  • Normally you manually configure all 4 pieces inside the ElevenLabs dashboard
  • With Claude Code you brainstorm at a high level, it researches the best approach, then writes the config + tools for you
  • Zero clicking through the ElevenLabs UI for setup

Tools & links:

  • ElevenLabs Agents — voice agent platform (STT, TTS, agent config, tools, KB)
  • Claude Code — the orchestrator doing all the heavy lifting
  • Firecrawl — scraping tool Nate frequently pairs with Claude Code via MCP server
  • AI Automation Society Plus (Skool) — Nate's paid course/community

Why it matters for you: This is the exact "natural language → working agent" pattern you want from Nate — concrete tools (ElevenLabs + Claude Code + Firecrawl/MCP), a clear mental model of voice agent internals, and a repeatable recipe you can swap your own knowledge base into. Anthropic Users Are PISSED

10.5K

Anthropic billed Max users extra for using rival harnesses.

If you build with Claude Code, a stray 'Hermes' or 'OpenClaw' string in your repo could quietly trigger overage charges on your Max plan.

details

What it is: A bug (or feature, depending who you ask) where Claude Code detected third-party harness names in git commits/repo strings and either refused requests or billed Max subscribers extra usage.

How it works:

  • Claude Code pulls `git status` into its system prompt
  • If strings like `Hermes` or `OpenClaw` appear in recent commits, the request gets flagged as using a non-Anthropic harness
  • Result: refusal to run, or surprise overage charges on top of $100/$200 Max plans
  • One user (per Dwarkesh Patel's repost) hit $200.98 in extra usage with 13% weekly / 0% session showing on the dashboard
  • Bisected commits, found `Hermes.md` was the trigger
  • Anthropic support initially called it an 'authentication routing issue' and refused refunds — citing policy against compensating for 'degraded service or technical errors'
  • Reversed course only after Theo Brown's and Patel's tweets hit 1M+ and 1.4M views
  • Tariq (Anthropic) confirmed it was a bug in third-party harness detection + git-status injection; affected users got refunds plus a month of credit

Tools & links:

  • Claude Code — Anthropic's official CLI harness
  • OpenClaw / Hermes — third-party Claude harnesses that triggered the detection

Why it matters for you: You ship web apps with Claude Code on a paid plan — audit your commit messages and repo strings, because Anthropic's harness-detection logic is silently reading your git state and can bill you for it.
