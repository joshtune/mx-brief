v2.1.114

Native binary, sandbox hardening, /loop & /ultrareview polish.

Big daily-driver upgrade: faster native binary, safer Bash rules, and sharper /loop, /ultrareview, and multiline editing — all things that hit you every session.

details

What changed:

  • CLI now spawns a native Claude Code binary per platform instead of bundled JS (faster startup)
  • New `sandbox.network.deniedDomains` setting overrides wildcard `allowedDomains`
  • `/loop`: Esc cancels pending wakeups; clearer "Claude resuming /loop wakeup" label
  • `/ultrareview`: parallelized checks, diffstat in launch dialog, animated launching state
  • `/extra-usage` and `@`-file autocomplete now work from Remote Control (mobile/web)
  • Multiline input: `Ctrl+A`/`Ctrl+E` jump to line start/end (readline parity); Windows `Ctrl+Backspace` deletes previous word
  • Fullscreen: `Shift+↑/↓` scrolls viewport while extending selection
  • Long wrapped URLs stay clickable (OSC 8 terminals)
  • Stalled subagents now fail after 10 min instead of hanging
  • Bash transcript shows full multi-line command even if first line is a comment (UI-spoofing fix)
  • `cd <cwd> && git …` no longer prompts for permission when `cd` is a no-op
  • Fixes: Cmd-backspace/`Ctrl+U`, markdown tables with pipe-containing code spans, `/copy` full-response table alignment, `/insights` EBUSY on Windows, emoji char-count in copy toast, session recap firing mid-compose, messages typed during subagent view being misattributed
  • v2.1.114 hotfix: permission dialog crash when agent-team teammate requested tool permission

Breaking changes:

  • Security tightening may reject previously-allowed commands:
  • `Bash(find:*)` allow rules no longer auto-approve `find -exec` or `-delete`
  • Bash deny rules now match commands wrapped in `env`/`sudo`/`watch`/`ionice`/`setsid`
  • macOS: `/private/{etc,var,tmp,home}` paths treated as dangerous under `Bash(rm:*)` allow rules
  • Bash `dangerouslyDisableSandbox` no longer skips permission prompt silently
  • Native binary distribution via per-platform optional dependency — may affect custom install/CI setups

Links:

  • v2.1.114 release notes
  • v2.1.113 release notes

Why it matters for you: As a daily Claude Code user, the native binary speeds up every session, `/loop` + `/ultrareview` improvements tighten your workflow, and the tightened Bash allow-rules mean you should audit your `.claude/settings.json` for any `find:*` or `rm:*` entries before they start behaving differently.
