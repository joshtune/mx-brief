Automating GitHub Issues with Claude Code and Coder Workspaces

TL;DR

Cole Medin demonstrates how to use Coder Workspaces — a free, open-source orchestration layer — to automatically spin up remote Claude Code instances that handle GitHub issues end-to-end, from reading the issue to creating a pull request, without anything running on your local machine.

The Problem: Running Coding Agents Locally Doesn't Scale

When using AI coding agents like Claude Code, each session typically runs on your own machine. This means you can only handle one task at a time, your local resources are tied up, and there's no easy way to share the infrastructure with a team. Cole Medin shows how Coder Workspaces solves this by providing a remote orchestration layer that manages multiple coding agent sessions in parallel.

What Is Coder Workspaces?

Workspaces is a free, open-source tool by the platform Coder. It is not a coding agent itself — it's infrastructure for orchestrating coding agents at scale remotely. Each coding agent session gets its own isolated environment, which means you can run dozens of issues in parallel without any of them interfering with each other. Cole has it self-hosted on a VPS (virtual private server), though he does not specify which VPS provider or how the self-hosting setup was done.

The Workflow: From GitHub Issue to Pull Request

The demonstrated workflow is straightforward. Cole opens a GitHub issue in his repository and adds a tag called "coder" to it. That's the only manual step. Once the tag is applied, Coder Workspaces detects the tagged issue and automatically spins up a remote environment. It registers the task in the Workspaces dashboard under the "tasks" section. A comment is automatically posted on the GitHub issue with a direct link to track the work as it happens in real time. Claude Code then runs inside the remote environment, reads the issue, implements the solution, and creates a pull request. The pull request is ready for human review and merge — Cole explicitly states he did not write a single line of the resulting PR.

Scaling: Parallel Issue Handling

The key scaling advantage emphasized is that because each agent session gets an isolated environment, you can process dozens of GitHub issues simultaneously. This infrastructure can also be shared across an entire team, so multiple developers can trigger and track coding agent tasks through the same Workspaces deployment.

Key Takeaways

  • Coder Workspaces is a free, open-source orchestration layer for coding agents — not an agent itself, but infrastructure to run agents like Claude Code remotely and in parallel
  • The trigger mechanism shown is simple: add a "coder" tag to a GitHub issue, and Workspaces automatically spins up a Claude Code instance to handle it
  • Each coding agent session runs in its own isolated environment, enabling parallel processing of many issues without local resource constraints
