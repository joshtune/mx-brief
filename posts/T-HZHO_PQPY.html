Setting Up OpenClaw: AI Agent Gateway on a VPS with Telegram Integration

TL;DR

OpenClaw is an open-source AI agent gateway that connects AI models (OpenAI, Anthropic, local) to messaging channels (Telegram, Discord, Slack) with persistent memory and system-level tool access. This tutorial walks through installing it on a VPS, configuring an agent with Telegram, building quick projects (news dashboard, IT monitoring), and locking down security.

What Is OpenClaw and Why Does It Matter?

OpenClaw is not itself an AI — it's a gateway (a Node.js service) that connects three things together: (1) any AI model you choose (OpenAI, Anthropic, local models via Ollama), (2) communication channels you already use (Telegram, Discord, Slack), and (3) a persistent memory system stored as markdown files on your server. At the time of recording, it had 308,000 GitHub stars, surpassing React and the Linux kernel. The creator was acqui-hired by OpenAI, and companies like Anthropic (with Channels in Dispatch) and Nvidia (with Nemo Claw) have built their own versions. Jensen Huang called OpenClaw 'the operating system for personal AI.' The key insight is that it packages existing capabilities — tool use, memory, multi-channel communication — into one clean install that makes AI agents feel accessible. You install it on a machine and essentially say 'this is your computer,' giving the agent access to bash, cron, file system, and more.

Prerequisites

You need a server to install the OpenClaw Gateway on. The video uses a Hostinger VPS (KVM2 tier specifically), but any Linux server with SSH access works. You also need a Telegram account (for the messaging channel), and either an OpenAI API key or a ChatGPT Pro (or higher) subscription. Local models via Ollama are also officially supported but the creator notes you'll have a better experience with OpenAI or Anthropic.

Step 1: Provision a Server and SSH In

After provisioning your VPS (the video uses Hostinger's KVM2 plan), copy the SSH command from your hosting dashboard, open your terminal, paste the SSH command, and connect to your server.

Step 2: Install OpenClaw

Go to openclaw.ai and find the one-liner install command. Copy it, paste it into your terminal on the server, and run it. The installation completes quickly. You'll see a security warning — the video says to scroll past it and hit 'Yes, I understand' for now (security hardening is covered later). Then select 'Quick Start.'

Step 3: Choose Your AI Model

The installer prompts you to choose which AI model to use as the 'brain.' Options include OpenAI, Anthropic, and local models (Ollama). The video recommends OpenAI. If you already pay for ChatGPT Pro or higher, you can use your existing subscription instead of a separate API key. Select the subscription option, and it walks you through an OAuth flow: you'll get a URL to open in your browser, log into your ChatGPT account, copy the redirect URL from your browser's address bar, paste it back into the terminal, and hit enter. When prompted for the model, the video goes with the default: GPT 5.4.

Step 4: Configure Telegram as Your Channel

When prompted to choose a communication channel, select Telegram. You need to create a Telegram bot: open Telegram, start a chat with @BotFather, send '/newbot', name your bot, give it a username (must end in 'bot'), and BotFather gives you a bot token. Copy the token, paste it into the terminal when prompted. For web search, the video selected Brave but skipped the API key by hitting enter. Skip configuring skills for now (select 'no'). For hooks, enable: boot, bootstrap, command logger, and session memory.

Step 5: Hatch Your Agent via TUI

Choose TUI (Terminal User Interface) instead of the web UI to launch your agent. The agent wakes up and you configure it by talking to it directly — this is a key aspect of OpenClaw. You tell it who it is and who you are. The video names the agent 'Terry Cruz' and describes the user as 'a Network Chuck fan, chaotic, muscles emoji.' Everything you say gets written to the agent's soul.md file. After the terminal setup, go to Telegram, find your bot, send 'Start.' There's a security handshake — not just anyone can talk to your agent out of the box. You need to sync it up by copying the configuration output from the terminal and pasting it to the agent in Telegram to finish configuring the channel.

Project 1: One-Shot News Briefing Dashboard

To demonstrate OpenClaw's capabilities, the video compares it to a previously complex N8N workflow that took an entire video to build. In Telegram, the creator simply told the agent: give me cybersecurity news, scrape Reddit, check Hacker News, include YouTube videos, rate whether each item is worth reading/watching. The agent completed the task autonomously. Then, after putting the agent in 'high thinking mode,' the creator asked it to build a dashboard — and it generated a live web dashboard with the news results, accessible via a DNS name on the server.

Project 2: Self-Monitoring IT Engineer Dashboard

The second demo turns the agent into an IT engineer monitoring its own server. Using '/new' for fresh context and enabling high thinking mode, the creator told the agent: 'You're an IT engineer. Your first job is to monitor the server you're on. Check everything. Internet speed, RAM, CPU, security, logs, and create a dashboard for me.' The agent investigated its own system, made sure it wouldn't break anything, then built a live-updating monitoring dashboard — all in one shot.

Under the Hood: OpenClaw's File Structure

OpenClaw is not magic — it's directories and markdown files. The gateway itself is a Node.js process you can see running. The main files live in the agent's workspace directory. The soul.md file contains the agent's personality and core identity. The identity file holds what you filled in during first conversation. The memory file is long-term memory (like a wife's birthday, favorite things). There's also a memory directory with daily journal files — one file per day logging conversations and events. The agents.md file is the master instruction prompt: it defines first-run bootstrap behavior, session handling, memory loading rules (e.g., long-term memory isn't loaded in group chats by default), red lines (security boundaries), and protocols for when to reach out vs. stay quiet. At the end of the day, it's 'just a fancy prompt' — and it's not tied to any provider. You can swap AI models and keep all the memory intact, essentially doing a 'brain transplant.'

ps aux | grep claw
cd ~/openclaw
ls
cd workspace
ls
cat soul.md
cat identity
cat memory
cd memory
ls
cat agents.md

Cron Jobs and Heartbeats: What Makes It Feel Alive

The creator emphasizes that crons and heartbeats are what make OpenClaw feel alive and differentiate it from competitors (Claude and others are trying to copy this). You can tell your agent to check in every hour, send a daily news briefing at 6 AM, or remind you to do things. Under the hood, these are real cron jobs on your server — nothing magical. In the demo, the creator told the agent 'every 30 minutes, remind me to drink coffee' and 'every hour, pop in to say hi.' The agent chose to implement both as cron jobs rather than heartbeats. You can view them in the cron directory or via the CLI.

openclaw cron list

Skills and ClawhHub

OpenClaw has a skills system with a directory called ClawhHub (clawhub) where you can browse over 33,000 community-contributed skills. Skills give your agent additional capabilities. The video demonstrates installing a Word document creation skill. ClawhHub partners with VirusTotal for scanning because 12% of skills were found to contain malware — be very careful and vet everything you install. After installing the Word skill, the agent was asked to create its own resume and produced a complete Word document.

npm -i -g clawhub
clawhub install word-docx

Additional Agent Capabilities

The agent has a headless browser built in — the creator told it to go to networkchuck.coffee and it browsed the site and even added a product to a cart. OpenClaw also supports sub-agents: you can deploy a sub-agent for a specific research task (the demo deployed one to research the best way to make coffee). In Telegram, you can use commands like '/status' to see active sessions and sub-agents, and '/subagents list' to manage them. You can also install multiple agents under one gateway — the creator has an IT department setup with Ron Ezy as IT manager and Fred, George, and Charlie as individual engineers (network, storage, systems), all accessible via Slack.

Security: Auditing Your Installation

The creator stresses that OpenClaw security was initially a 'dumpster fire' but has improved significantly. The first thing to do after setup is run the security audit command. This analyzes your instance against security best practices. A fresh install following the video's steps should show only two or three warnings, nothing critical. You can also run the auto-fix variant, though be careful as it might break things. The most important check is ensuring your web UI is not exposed to the internet. By default, OpenClaw binds to 127.0.0.1 (loopback), meaning only processes on the host can access it. The web UI runs on port 18789.

openclaw security audit
openclaw security audit --fix

Accessing the Web UI via SSH Tunnel

Since the web UI is bound to localhost, you access it from your local machine via an SSH tunnel rather than exposing it to the internet. Run the SSH tunnel command from a new terminal window on your local machine (not the one logged into the server), substituting your username and IP. Then browse to localhost:18789. If prompted for a gateway token, run 'openclaw config' on the server, navigate to Gateway settings, and find/generate your token. The web UI shows agent chat, sessions, instances, channels (Telegram, Discord), and cron jobs.

openclaw config

Firewall Configuration

Verify your firewall is enabled and properly configured. The video enables the firewall allowing only SSH (port 22) by default, blocking everything else. When the firewall was enabled, the dashboards the agent had built became inaccessible until port 8787 was explicitly allowed. The creator actually had the agent itself open the firewall port — demonstrating the approval flow where you must confirm the agent's actions.

Security: Tools Profile and Execution Permissions

Two key security configurations control what your agent can do. The tools.profile setting controls which tools the agent can see: 'full' means every tool on the system is visible; 'coding' is limited to read/write files and terminal commands (no browsing or web search). The tools.exec.security setting controls what the agent can do with tools it knows about: 'full' means unrestricted execution, 'allow-list' restricts to a configured list, 'deny' blocks everything. There's also a tools.exec.ask setting: 'off' means never ask, 'always' means always ask before using any tool, 'only on miss' means only ask when the tool isn't on the allow list. These settings are global across all sessions. Per-session configuration is possible in Telegram but not covered in this video. The full configuration can be viewed in the openclaw.json file.

openclaw config get tools.profile
openclaw config set tools.profile full
openclaw gateway restart
openclaw config get tools.exec
openclaw config set tools.exec.security allow-list
openclaw config set tools.exec.ask off
cat openclaw.json

Red Lines, Yellow Lines, and Always-Allowed Actions

After giving the agent broad permissions, you constrain it using the red lines system in the agents.md file. Red lines are hard stops — things the agent must never do without asking (e.g., don't exfiltrate private data, don't run destructive commands, don't modify SSH config). Yellow lines are actions the agent can take but must log (e.g., firewall changes, running Docker commands). Always-allowed actions are things the agent can do freely. Important caveat: these are prompt-based instructions, not deterministic enforcement. There is nothing technically preventing the agent from violating red lines — it's an instruction the AI model should follow, not a hard security boundary.

Key Takeaways

  • OpenClaw is a gateway/harness, not an AI itself — it connects any AI model to messaging channels with persistent memory and system tool access, all running as a Node.js service on your server
  • Installation is a one-liner from openclaw.ai, and you can use your existing ChatGPT Pro subscription instead of a separate API key
  • Everything is stored as markdown files (soul.md, identity, memory, daily journals, agents.md) — you can inspect and manually edit anything the agent does
  • Run 'openclaw security audit' immediately after installation to check for security issues; ensure the web UI (port 18789) is not exposed to the public internet
  • 12% of ClawhHub skills were found to contain malware — vet every skill before installing
  • Red lines in agents.md are prompt-based constraints, not deterministic enforcement — there's no hard technical barrier preventing the agent from violating them
  • Cron jobs and heartbeats are what differentiate OpenClaw from standard AI assistants — they're real scheduled tasks on your server that make the agent proactively useful
  • The tools.profile (what tools the agent can see) and tools.exec.security (what it can do with them) are the two most important security configurations to understand
