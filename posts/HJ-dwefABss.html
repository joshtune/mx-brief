Setting Up Paperclip: Open-Source AI Agent Orchestration with Claude Code

TL;DR

Paperclip is a free, open-source (MIT license) orchestration platform for running multiple AI agents as a virtual company — complete with a dashboard, ticketing system, heartbeats, budgets, and approval workflows. This tutorial walks through setting it up from scratch with Claude Code, creating agents, assigning tasks, installing skills, and managing an autonomous agent organization.

What Is Paperclip and Why It Blew Up

Paperclip is a free, open-source orchestration tool for building "zero human companies" — organizations staffed entirely by AI agents. It launched in early March 2026 and had already hit over 36,000 GitHub stars within about three weeks. It carries an MIT license.

The creator explains that one of Paperclip's founders, Dota, described the core pain point: running 20 different terminal sessions with 20 different Claude Code instances, then coming back and having no visibility into what each one was doing or what you'd asked them to do. Paperclip solves this by providing a centralized dashboard that keeps all agent sessions synchronized, with each session understanding the overall goal being worked toward.

Paperclip supports multiple providers and tools — you can use Claude Code, OpenClaw, Codex, Cursor, or bring your own agents. Each agent can use a different model and provider. By default it runs on localhost, but it can be deployed to a VPS for remote access.

Dashboard Overview and Core Concepts

The Paperclip dashboard provides a mission-control-style view with several key areas:

- **Agents panel**: Shows all enabled agents and what they're currently working on. In the demo, seven agents were running across an existing company.

- **Tasks panel**: Shows all tasks and their status (in progress, completed, pending).

- **Inbox**: Displays items requiring human approval or review — this is how agents escalate decisions to you.

- **Activity feed**: Logs all recent activity across agents.

- **Issues/Ticketing**: A full ticketing system where every conversation and issue is logged. You can open issues, leave comments, and see what agents are thinking and doing.

- **Projects**: Can optionally be synced to a GitHub repo for code-producing agents.

- **Multiple companies**: You can run multiple companies inside a single Paperclip instance. Each company could represent a new startup idea, or different departments within an existing business.

The key mental model is that you act as the "board" — giving high-level goals, metrics, and desired outcomes — while the AI agents act as operators who figure out the implementation details.

Installation and Initial Setup

Setting up Paperclip is straightforward. Go to paperclip.ing, click "Get Started," and it gives you a single command to run in your terminal. You don't need to stay in the terminal after that — the command launches a server that opens a localhost dashboard in your browser.

Prerequisites: You need Claude Code already authenticated on your machine. If you're already authenticated into Claude Code somewhere on your terminal, Paperclip will be able to connect to it immediately.

Creating a New Company from Scratch

The onboarding flow walks you through company creation step by step:

1. **Name your company** — In the demo, the company was called "Proofshot" with the mission of letting customers record testimonials via a link, where AI cleans them up for embedding on websites.

2. **Create your first agent** — This doesn't have to be a CEO, but the demo starts with one. You select a model (Sonnet 4.6 was chosen) and run a quick connectivity test to verify the agent can hit the Claude Code CLI and respond.

3. **Set the first task** — The default first task is "Hire your first engineer and create a hiring plan," assigned to the CEO. You can add background context about your business here, or change the initial task entirely.

4. **Launch** — Click to start the company. The CEO agent begins running immediately, creating a plan and executing commands just like working in Claude Code directly.

Agent Configuration: Instructions, Soul, Heartbeat, and Tools

Each agent has four configuration files that define its behavior:

- **Soul file**: Defines who the agent is and how it should act. For the CEO, this included directives like "you own the P&L," "you default to action," "you hold the long view while executing the near term," and "you protect focus hard." You can customize voice, tone, and persona.

- **Heartbeat file**: An execution and extraction checklist that runs on every heartbeat cycle.

- **Tools file**: Defines what tools the agent has access to.

- **Agents file**: Information about other agents in the organization.

These files evolve over time and can be manually edited. All agents come pre-loaded with Paperclip skills, so they natively understand how Paperclip works — its API, protocols, and how to interact with the system.

Heartbeats: Keeping Agents Running Autonomously

Heartbeats are a core feature that keeps agents proactive. Agents wake up on a configurable schedule — every 4 hours, 8 hours, or 12 hours. When they wake up, they start with fresh context and fresh memory. This is why the configuration files are critical: on each heartbeat, the agent checks its work, looks at its tasks, and gets re-familiarized with its environment before continuing.

The creator noted that native heartbeats were one of the things that made OpenClaw blow up initially, because they made agents feel much more proactive rather than waiting for human input.

Hiring Agents and the Approval Workflow

When the CEO agent was given the task to hire an engineer, it produced a hiring request that appeared in the inbox for board approval. The request specified a "Founding Full Stack Engineer" with capabilities in integrations, design, database, and front end.

After approving the hire, the CEO automatically set up the new engineer agent with minimal initial instructions (just Paperclip skills) and then created five tasks for it. The CEO broke the work into milestones: MVP core back end, AI processing pipeline, front-end recording UI, embeddable testimonial card, and more.

By default, Paperclip requires board approval for new hires. This can be toggled off in Settings under "require board approval for new hires," which would let the CEO autonomously hire and grow the team.

The creator then created a follow-up issue for the CEO: hire a QA agent to ensure code quality, safety, and bug-free output. The instruction was written as a high-level goal — "continuously build, make improvements and refinements until you come back to me and say the QA agent and founding engineer are done" — demonstrating the board-level interaction pattern.

Projects and GitHub Integration

Under the Projects section, you can create projects and optionally sync them to a GitHub repo. The creator set up a project called "Initial MVP" described as "getting our first version of the website up and running so that we can continuously improve and refine." Projects can have a status, goal, and target date. Issues can then be assigned to specific projects for organization.

Installing Skills from skills.sh

Skills can be added at the company level or per individual agent. To install a skill, you paste a GitHub URL or path into the skills section of the dashboard.

The creator demonstrated installing a "front end design" skill by copying its GitHub URL from skills.sh (a marketplace of mostly free agent skills) and pasting it into the Paperclip dashboard. A second skill, "web design guidelines," was added the same way.

The creator warned that while skills.sh does have security audits, you should still be careful and exercise judgment about what you install — "you never ultimately know at the end of the day."

Routines (Beta): Recurring Automated Workflows

Routines allow you to define recurring workflows. The creator demonstrated setting up a nightly security check routine — an agent would run code checks and integration reviews daily to ensure no secrets are exposed and no cybersecurity vulnerabilities exist.

Routines can be assigned to specific agents, toggled active/inactive, and run on a configurable schedule. After creation, Paperclip takes you to a trigger setup for schedules, webhooks, or terminal runs. Note: you can only assign routines to agents that already exist (the creator couldn't assign to the QA agent because it hadn't been approved yet).

Budget Controls

Each agent can have a per-month budget set in the configuration. This lets you control exactly how much each agent spends. The creator noted that spend analytics only show up when using pay-per-token billing — if you're using a subscription (as the creator was), the dashboard shows zero spend.

Importing Pre-Built Company Templates

Paperclip has a growing catalog of pre-built company templates in its GitHub repo under a "companies" section. These are ready-to-deploy agent organizations that come with pre-configured agents, skills, and knowledge.

Examples mentioned:

- **G Stack**: Comes with a CEO, CTO, QA engineer, release engineer, staff engineer, and pre-loaded skills.

- **Agency agents**: For running an AI agency.

- **Scientific research company**: Specialized agents with specific domain knowledge (agents named after actual doctors).

- One template had 48 agents pre-configured.

The creator compared importing templates to a company acquisition — you bring in an existing team with established frameworks rather than building from zero.

Using Claude Code as a Paperclip Configuration Partner

The creator set up a dedicated Claude Code project specifically to help manage and configure Paperclip. This project was given knowledge of the full Paperclip architecture, API, heartbeat protocol, agent configurations, secrets management, and VPS migration planning.

This Claude Code project served as a "partner in crime" for:

- Creating and managing secrets/environment variables (Paperclip does have a built-in mechanism for environment variables, but it's not exposed in the UI — Claude Code knew how to access and configure it)

- Querying the state of companies and agents

- Monitoring and expanding the agent organization

- Understanding gotchas and troubleshooting

The creator recommended feeding Claude Code the Paperclip GitHub repo (since it's open source), letting it research how people are using it on X/Twitter, and building up its knowledge base. This is especially valuable during initial setup and orientation, even if you rely on it less over time.

Key Takeaways

  • Paperclip is a free, MIT-licensed, open-source tool for orchestrating multiple Claude Code (or other AI) agents from a single dashboard — install it with one terminal command from paperclip.ing
  • Act as the board, not the operator: give agents high-level goals and metrics, let them figure out implementation details, hiring, and task breakdown
  • Each agent has four config files (soul, heartbeat, tools, agents) that define its persona and behavior — these are critical because agents wake up with fresh context on each heartbeat
  • Heartbeats keep agents running autonomously on a schedule (4/8/12 hours) — they wake up, check their work and tasks, and continue without human prompting
  • Set per-agent budgets to control costs, and use the approval workflow (togglable) to maintain human oversight on hiring decisions
  • Install skills from skills.sh (a free marketplace) via GitHub URLs, but exercise caution and review what you're installing
  • Set up a dedicated Claude Code project to help you configure and manage Paperclip — feed it the open-source repo so it understands the full architecture and can help with secrets, monitoring, and troubleshooting
  • Import pre-built company templates from Paperclip's GitHub repo to start with a functional team rather than building from scratch
