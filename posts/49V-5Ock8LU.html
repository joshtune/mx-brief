18 Claude Code Token Management Hacks to Stretch Your Usage 2-5x

TL;DR

Claude Code's token system compounds exponentially per message because it rereads the entire conversation history every turn — 98.5% of tokens in a 100+ message chat were just rereading old history. This video covers 18 hacks organized in three tiers (basic, intermediate, advanced) to dramatically reduce token waste and extend your session life.

How Claude Code Tokens Actually Work

Before diving into the hacks, it's critical to understand how Claude Code charges tokens. A token is the smallest unit of text the model reads and charges for — roughly one token per word, though not exactly.

The key insight: every time you send a message, Claude rereads the entire conversation from the beginning. Message one, its reply, message two, its reply, all the way up to your latest prompt. This means your cost is compounding, not just adding — it's exponentially growing. Message one might cost 500 tokens, but message 30 costs 15,000 because it's rereading everything before it.

One developer tracked a 100+ message chat and found that 98.5% of all tokens were spent just rereading old chat history. After 30 messages, you might already be at almost a quarter million cumulative tokens.

On top of your own messages, Claude also reloads your CLAUDE.md, MCP servers, system prompts, skills, and files on every single turn. This is invisible overhead constantly dripping into your context.

Bloated context doesn't just cost more — it produces worse output. There's a phenomenon called "loss in the middle" where models pay the most attention to the beginning and end of a session, and content in the middle gets ignored. So you're paying more and getting less.

Tier 1: Easy Wins Everyone Should Implement

These nine hacks are straightforward to implement and provide immediate benefit.

Hack 1: Start Fresh Conversations

Use `/clear` between unrelated tasks. Don't carry context about topic A into a conversation about topic B. Every single message in a long chat is exponentially more expensive than the same message in a fresh chat. This one habit is the number one thing that extends your session life.

/clear

Hack 2: Disconnect MCP Servers

Every connected MCP server loads all of its tool definitions into your context on every message. This is invisible token drain — one server alone might consume around 18,000 tokens per message.

Run `/mcp` at the start of each session and disconnect the ones you don't need. Better yet, if you can find CLIs for something, use those instead. For example, rather than having the Google Workspace or Google Calendar MCP server (which eats a lot of tokens), just use the Google Workspace CLI. It's faster and cheaper. The creator suggests the future is moving towards agents using CLIs rather than MCPs.

/mcp

Hack 3: Batch Prompts Into One Message

Three separate messages cost three times what one combined message costs because of how token reprocessing works. Instead of sending "summarize this" as one message, then "now extract the issues," then "now suggest a fix" — send it all in one prompt.

If Claude did something slightly wrong, edit your original message and regenerate instead of sending a follow-up correction. Follow-ups stack onto history permanently, while edits replace the bad exchange entirely.

The creator does note there's an argument that giving AI one specific task at a time might produce better output quality since it's more specialized and focused. It depends on the use case, but you should be aware of the token tradeoff.

Hack 4: Use Plan Mode Before Any Real Task

Plan mode lets Claude map out the approach and ask you the right questions before writing code. This prevents the single biggest source of token waste: Claude going down the wrong path, writing code, and then having to scrap and redo everything.

The creator recommends adding a rule like this to your CLAUDE.md file:

Do not make any changes until you have 95% confidence in what you need to build. Ask me follow-up questions until you reach that confidence level.

Hack 5: Run /context and /cost

`/context` shows you exactly what's eating your tokens right now — conversation history, MCP overhead, loaded files, etc. `/cost` shows your actual token usage and estimated spend for the current session.

The creator ran `/context` in a completely fresh session with no chats and was already down 51,000 tokens before even talking to Claude, due to system prompt, system tools, custom agents, skills, and memory files. MCP servers were cleared in this example, but they can blow up your tokens from the start.

/context
/cost

Hack 6: Set Up a Status Line

The status line gives you real-time visibility in your terminal. The creator's setup shows the model being used, a visual progress bar of usage, context percentage, and token count (e.g., 52,000 tokens out of 1,000,000).

Note: this is your context window usage, not your 5-hour session limit. To set it up, run the `/status line` command in Claude Code and explain that you want it to show your model, context percentage, and token count.

/status line

Hack 7: Keep Your Dashboard Open

Keep the Claude usage dashboard pulled up so you can check your usage every 20-40 minutes and pace yourself. You could even set up an automation to check every 30 minutes and send you a text or Slack message when you're getting near your usage limit.

Hack 8: Be Smart With Pasting

Before dropping a document or file into Claude, ask: does Claude need to read this whole thing? If the bug is in one function, paste just that function. If it just needs the context of one paragraph, paste only that. Claude needs to be precise about what it reads, but you also need to be precise about what you feed it.

Hack 9: Watch Claude Code Work

Don't just fire off a prompt and walk away or switch tabs. Watch what Claude is doing, especially on longer tasks. Sometimes it goes down the wrong path or gets stuck in loops rereading the same files. If you catch it, stop it immediately. In a bad loop, 80% of tokens are being used producing zero value. Watch your session run until you know it's going down the right path — it could save you thousands of tokens.

Tier 2: Intermediate Optimizations

These five hacks require a bit more setup but deliver significant savings.

Hack 10: Keep Your CLAUDE.md File Lean

Claude auto-reads your CLAUDE.md at the start of every single chat as system context — not just per session, but per message. If your CLAUDE.md is 1,000 lines, the whole thing gets read every time you send a message, even if you just say "hi."

Keep it under 200 lines. Include only your tech stack, coding conventions, build commands, the 95% confidence rule, and the most important things. Treat it like an index that points to where more data lives. It can reference large files, but this way Claude only reads what it needs and knows exactly where to look when it does need something.

This is a mindset shift that applies beyond CLAUDE.md — use it with skills, master reference guides, etc. The creator mentioned someone who created a lean index file pointing Claude to specific URLs in Claude Code documentation, so it doesn't have to search through the whole database.

Hack 11: Be Surgical With File References

Don't say "here's my whole repo, go find the bug." Instead, say something like "check the verifyUser function inside the auth.js file." You can also use `@filename` to point at specific files instead of letting Claude explore freely. The whole idea is being specific and routing Claude directly to what it needs.

Hack 12: Compact at Around 60% Capacity

Auto-compact triggers at around 95%, by which point your context is already pretty degraded. Instead, run `/context` to check your capacity percentage (or use the status line), and at about 60%, run `/compact` with specific instructions on what it should preserve.

After three to four compacts in a row, quality starts to degrade. At that point, get a session summary, run `/clear`, give the session summary back to a fresh session, and keep going.

/compact
/context
/clear

Hack 13: Short Breaks Are Actually Costing You

Claude Code uses prompt caching to avoid reprocessing unchanged context, but the cache has a 5-minute timeout. If you step away for longer than 5 minutes, your next message reprocesses everything from scratch at full cost. This is why some people feel like their usage randomly spikes after stepping away.

If you're going to take a break, consider running `/compact` or `/clear` before you step away.

Hack 14: Command Output Bloat

When Claude runs shell commands, the full output enters your context window. If a command returns 200 commits or tons of data, all of that becomes tokens sent to the model. This is invisible because when Claude runs a bash command, you often just see one line — not the full token cost of the output.

Be intentional about what you let Claude run. If you know a project doesn't need certain commands, deny those permissions in that project.

Tier 3: Advanced Strategies

These four hacks are for power users looking to squeeze maximum value from their allocation.

Hack 15: Pick the Right Model

Use different models for different tasks:

- **Sonnet**: Default for most coding work

- **Haiku**: Sub-agents, formatting, simple tasks

- **Opus**: Deep architectural planning, only when Sonnet wasn't enough — try to keep Opus under 20% of usage

For large codebases where you want to do reviews, consider bringing in Codex (there's an official plugin now). You can have Opus and Sonnet working together to build a project, then bring in Codex to review everything, saving Claude tokens.

Hack 16: Understand the Cost of Sub-Agents

Agent workflows use roughly 7-10x more tokens than a standard single-agent session because each sub-agent wakes up with its own full context — a separate instance that reloads everything (files, system tools, etc.).

However, you can strategically delegate to sub-agents for one-off tasks, especially using Haiku. If you need to process a lot of information or do extensive research and just get a summary back, having 80% of your tokens go to a cheaper model (Haiku) rather than an expensive model saves money. Agent teams can produce higher quality output, but they're very expensive — use them sparingly.

Hack 17: Understand Peak Hours

Anthropic adjusts how fast your 5-hour session window drains based on demand. Peak hours are 8 AM to 2 PM Eastern time on weekdays. Off-peak hours (afternoons, evenings, weekends) give you normal or extended usage.

Strategically schedule big refactors, multi-agent sessions, or big projects during off-peak hours only.

**Hack 17.5**: If you're near a reset and have room left in your allocation, go heavy — try to hit the usage limit before it resets. Let your agents go loose. Conversely, if you're near your limit but have lots of time remaining, step away. Take a break, come back with a full budget instead of burning the last 5% on something small and getting stuck mid-task.

Hack 18: Your CLAUDE.md as a System Constitution

Your CLAUDE.md should contain stable decisions, architecture rules, and progress summaries. Think of it as the source of truth that makes every prompt shorter. Save decisions, not conversations — every architectural call you store there is a paragraph you never have to type again.

You can add rules that optimize token usage directly, such as instructing Claude to use sub-agents for exploration or research, and to spawn them using Haiku.

The creator shared a prompt from the bottom of their CLAUDE.md for self-learning. Note: you have to check on this frequently because you don't want it to accidentally get too bloated.

Use sub-agents for any exploration or research. If a task needs three plus files or multi-file analysis, spawn a sub-agent and only return summarized insights. Spawn that sub-agent in haiku.

Final Thoughts: Quality vs. Cost

The creator emphasizes two closing points. First, there's a balance between quality and cost — sometimes you need the higher quality option, which costs more, and that's the tradeoff.

Second, keep it simple. Most people don't need a bigger plan — they need to stop resending their entire conversation history 30 times when they could send it 5 times. It's not a limits problem, it's a context hygiene problem.

He also reframes hitting your limit: if you're applying these hacks and not being wasteful, hitting your limit means you're a power user getting massive leverage from the tool. People using it that much will be far more productive than those who never hit their limits.

Key Takeaways

  • Claude rereads your entire conversation every message — cost compounds exponentially, not linearly. One developer found 98.5% of tokens in a 100+ message chat were just rereading history.
  • Use /clear between unrelated tasks. This is the single highest-impact habit for extending session life.
  • A single MCP server can add ~18,000 tokens per message of invisible overhead. Run /mcp and disconnect what you don't need.
  • Add a 95% confidence rule to your CLAUDE.md so Claude plans before coding, preventing the biggest token waste: going down the wrong path.
  • Run /context and /cost regularly. The creator had 51,000 tokens consumed before even sending a message in a fresh session.
  • Keep CLAUDE.md under 200 lines and treat it as an index that points to files, not a dump of all project info.
  • Compact manually at 60% context capacity — auto-compact at 95% is too late and context is already degraded.
  • Prompt cache expires after 5 minutes — stepping away means your next message reprocesses everything at full cost.
  • Use the right model per task: Sonnet for coding, Haiku for sub-agents and simple tasks, Opus sparingly for architecture.
  • Schedule heavy work (refactors, multi-agent sessions) during off-peak hours: after 2 PM Eastern on weekdays, or on weekends.
