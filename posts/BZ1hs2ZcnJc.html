Anthropic's Claude Code Leak Reveals the Post-Prompting Era: AI That Acts Without Being Asked

TL;DR

A leaked Claude Code source revealed Anthropic is building 'Chyros,' an always-on daemon that autonomously fixes bugs, responds to messages, and monitors your GitHub — signaling a shift toward proactive AI that works in the background without prompts. This week also saw OpenAI's record $122B raise with plans for a 'super app,' new open-weight LLMs approaching frontier quality, and Microsoft's best-in-class transcription model.

The Claude Code Leak: What Was Actually Found

The biggest news of the week was the leak of Anthropic's Claude Code source code. According to Matt Wolfe, the original discovery was made by a user called "Fried Rice" on X, who found that Claude Code's source was exposed via a map file in their npm registry. Anthropic — described as a company that typically keeps things close to their chest — has been issuing DMCA takedowns as the code spread across GitHub, but the most interesting aspect isn't the leak itself; it's what people found inside.

A VentureBeat article (cited as the best breakdown) revealed a sophisticated three-layer memory architecture. At its core is a file called memory.md — a lightweight index of pointers that is perpetually loaded into context. Critically, this index doesn't store data; it stores locations. Raw transcripts are never fully read back into context — instead they're grepped for specific identifiers. Wolfe's interpretation: as you talk to Claude Code, it saves information and references in a memory file, and that memory file tells the context where to search for relevant information.

Chyros: The Always-On Background Agent

The most significant finding, in Wolfe's opinion, was something called Chyros — described in the leaked code as an 'autonomous daemon mode.' A daemon is a process that runs in the background of your computer, waiting for triggers. Chyros would allow Claude Code to operate as an always-on background agent.

Wolfe cited an X post that he said did the best job explaining it: Chyros runs in the background 24/7 while you work or sleep. Every few seconds, it receives a 'heartbeat' — essentially a prompt that asks 'anything worth doing right now?' It evaluates the current state and either acts or stays quiet. When it acts, it can fix code errors, respond to messages, update files, or run tasks — anything Claude Code can already do, just without the user initiating it.

Chyros has three exclusive tools that regular Claude Code doesn't get: (1) Push notifications — it can reach you on your phone or desktop even when you're not in the terminal. (2) File delivery — it can send you things it created without you asking. (3) Pull request subscriptions — it can watch your GitHub and react to code changes autonomously. It also keeps daily logs of everything: what it noticed, what it decided, and what it did.

Examples given: Your website goes down while you're asleep — Chyros detects it, restarts the server, sends you a notification, and by the time you see it, it's already resolved. A customer complaint email arrives at 2 AM — Chyros reads it, sends the reply, and logs what it did. Your Stripe subscription page has a typo that's been live for 3 days — Chyros spots it, fixes it, and logs the change.

Wolfe emphasized that Chyros is NOT yet rolled out into Claude Code — it was only discovered in the source code. But he sees it as evidence of a direction he's been discussing publicly, including at a CES panel: a 'post-prompting era' where AI becomes proactive, learns what you do and what you need, and acts on your behalf without explicit prompts. He noted that OpenAI is making similar moves with cron jobs and autonomous heartbeat patterns in their tools.

Other Leak Findings: Capiara, Mythos, and a Tamagotchi

People also found more evidence of 'Capiara' (discussed in a previous video), which appears to be used interchangeably with 'Mythos' — described as the next level of model for Anthropic. Additionally, a 'hidden buddy system' was discovered: a Tamagotchi-style terminal pet with stats like 'chaos' and 'snark.' Wolfe noted that many people are speculating this was a planned April Fool's joke that got sidelined because of the leak.

Boris Churnney, described as the head of Claude Code, publicly addressed the leak, calling it a developer error. His statement: 'Mistakes happen. As a team, the important thing is to recognize it's never an individual's fault. It's the process, the culture, or the infrastructure.' He explained there was a manual deploy step that should have been better automated, and the team has already made improvements. Wolfe noted that no customer data was leaked, Anthropic hasn't made a big public statement about it, and nobody appears to have been fired. However, Wolfe predicted spin-off versions of Claude Code will likely appear on the market soon from people leveraging the leaked source.

OpenAI's Record $122 Billion Raise and the 'Super App' Vision

OpenAI raised $122 billion at a valuation of $852 billion — described as the largest fundraising raise by any company ever, making OpenAI the fastest-growing company in history. From their own blog post: 'At this stage, we are growing revenue four times faster than the companies who defined the internet and mobile eras, including Alphabet and Meta.' They are now generating $2 billion in revenue per month. Notably, Microsoft participated in this round despite widespread speculation about a falling out between the two companies.

Buried at the bottom of the fundraising announcement, Wolfe found what he considers the most interesting part — OpenAI's roadmap. They stated: 'We're building a unified AI super app. As models become more capable, the limiting factors shift from intelligence to usability. Users do not want disconnected tools. They want a single system that can understand intent, take action, and operate across applications, data, and workflows. Our super app will bring together chat, GPTs, Codex, browsing, and our broader agentic capabilities into one agent-first experience.'

Wolfe drew a parallel to Anthropic's existing approach, where Claude's single app already integrates Claude chat, Claude Co-work, and Claude Code. He speculated that OpenAI is heading in this same direction and wouldn't be surprised if they also follow Anthropic's lead on proactive background agents — especially since they brought on Peter Steinberger, the creator of OpenClaw, to the OpenAI team. This is also cited as the reason they phased out Sora and appear to be skipping a planned 'adult mode,' instead focusing on core competencies: intelligent models for chat, research, coding, and agentic use cases.

Sora Was Losing $1 Million Per Day

A Wall Street Journal article revealed that Sora, OpenAI's video generation model that was shut down the prior week, was losing roughly $1 million per day — over a third of a billion dollars per year. Wolfe noted that with that context, the shutdown makes obvious sense.

Microsoft MAI-Transcribe-1: Best-in-Class Speech Recognition

Microsoft released MAI-Transcribe-1, a new speech recognition model achieving best-in-class accuracy across 25 languages, purpose-built for transcription workloads. On the WER (Word Error Rate) benchmark, it significantly outperforms GPT Transcribe, Scribe V2, Gemini 3.1 Flash, and OpenAI's Whisper model. It also reportedly works well in noisy environments.

Wolfe tested it with early access in the MAI playground, using sentences loaded with homophones (words that sound the same but have different meanings — like 'see' vs. 'sea,' 'here' vs. 'hear'). The model correctly distinguished between all the homophones, which Wolfe attributed to its contextual understanding. He also noted that MAI-Transcribe-1, along with MAI Voice One and MAI Image 2, are now available inside Microsoft Foundry for developers to build on top of.

Google Veo 3.1 Lite and Price Drops

Google rolled out Veo 3.1 Lite, a more cost-effective video generation model. Pricing: Veo 3.1 standard is about 40 cents per video with audio, Veo 3.1 Fast is 15 cents, and Veo 3.1 Lite is only 5 cents for 720p video. The Lite model does not support 4K generation — you need the standard or Fast tier for that. Logan Kilpatrick from Google announced that prices for Veo 3.1 Fast were also being reduced on April 7th. Wolfe speculated (without insider info) that there may be additional Google announcements around that date.

New LLM Releases: Open-Weight Models Closing the Gap

Several new large language models dropped this week:

**Gemma 4 (Google):** An open-weight model released under Apache 2.0, designed to run on Android devices and laptop GPUs. For users with more powerful hardware like an Nvidia Spark or Apple Mac Studio with decent memory, it's also designed to run agentic workflows (e.g., OpenClaw running locally). Benchmarks were only compared to past Gemma models but showed strong results.

**Qwen 3.5 Omni (Alibaba):** An omnimodal model handling text, image, audio, and video understanding. A standout feature: 'audio-visual vibe coding' where you describe your vision to the camera and it builds a functional website or game. Claims to outperform Gemini 3.1 Pro in audio and match its audio-visual understanding.

**Qwen 3.6 Plus (Alibaba):** Released just days after the 3.5 Omni, this model targets real-world agents with enhanced agentic coding, multimodal vision, and a 1 million token context window available by default. Benchmarks compare it to Claude 4.5 Opus (not the newest Opus) and Gemini 3 Pro (not 3.1). On Terminal Bench it beats those models; on SWE-Bench Pro it's roughly on par with Opus 4.5. This model is not open-weight, though Alibaba mentioned plans to open-source more models soon.

**Arcee Trinity Large Thinking:** From a company called Arcee (Wolfe's first time hearing of them), this is an open-source model under Apache 2.0. Benchmarks compare it to Opus 4.6, Kimmy K 2.5, GLM5, and Minimax M2.7 — roughly on par but not clearly beating them.

Wolfe's broader observation: open-weight models keep gaining ground on closed-source frontier models. The closed-source models always seem 'a few steps further,' but the gap is shrinking. He also candidly admitted he struggles to test new LLMs because existing models already handle his use cases well, and expressed interest in creating an 'everyday user benchmark' — inviting collaboration via X or LinkedIn.

Rapid Fire: Claude Computer Use, Codex in Claude Code, ChatGPT in CarPlay, and More

**Computer Use in Claude Code:** Anthropic's computer use feature (mouse and keyboard control in the browser) has been added to Claude Code, available in preview for Pro or Max plan subscribers.

**Codex Plugin for Claude Code:** A Codex plugin now exists for Claude Code, allowing users to bring their OpenAI key into Claude Code's workflow so Codex can review code and delegate tasks. Wolfe expressed surprise that Anthropic would allow this.

**ChatGPT in Apple CarPlay:** ChatGPT is now available inside Apple CarPlay for in-vehicle conversations.

**OpenAI Acquires TBPN:** OpenAI acquired TBPN (Tech Business Production Network), described as 'the ESPN of tech/business' — a daily three-hour live show on X and YouTube featuring founder interviews and news. Wolfe found this surprising given OpenAI's stated focus on avoiding 'side quests,' and raised editorial independence questions: Can they still joke about OpenAI? Can they interview competitors like Anthropic and Google?

**Perplexity Computer for Taxes:** Perplexity released a tax-focused use case for their computer model that can help draft federal income tax returns on official IRS forms, review professionally prepared returns, and build dashboards for complex tax scenarios.

**Salesforce Making Slack More Agentic:** 30 new capabilities announced for Slackbot, including meeting transcription and note-taking (even for Zoom meetings), reusable AI skills, MCP client functionality, native customer management, deep research, and voice input. Currently on Business Plus and Enterprise Plus plans, with free and pro plans getting access starting in April.

**GM Using AI for Car Design:** GM is using AI to transform hand-drawn sketches into concept videos, suggest design changes, and run early-stage aerodynamic testing — dramatically compressing the design timeline.

**Instacart AI Shopping Carts:** Instacart introduced physical AI-powered smart carts with touchscreens, outward-facing cameras, basket cameras, location sensors, and certified scales — all powered by Nvidia Jetson. These process what's happening in real time to influence shopping while it's happening. Wolfe was skeptical about consumer reception, questioning whether shoppers actually want product recommendations from their cart.

The Post-Prompting Thesis: Where All of This Is Heading

The throughline Wolfe draws across this week's news is the shift toward what he calls the 'post-prompting era.' Anthropic's Chyros daemon, OpenAI's super app vision with agentic capabilities, cron jobs, and the hiring of OpenClaw's creator — all point in the same direction: LLMs are moving into the background to become 'plumbing' rather than the primary interface users interact with.

As Wolfe put it: 'I don't want to have to sit and have conversations with it. I want it to know what it's supposed to be doing and just go do the things it's supposed to be doing.' The vision is AI that learns your workflows, monitors your systems, and acts proactively — notifying you after the fact rather than waiting for instructions. This is corroborated by OpenAI's own words: 'the limiting factors shift from intelligence to usability.'

Key Takeaways

  • Anthropic's leaked 'Chyros' daemon reveals a near-future where Claude Code runs autonomously in the background — monitoring GitHub PRs, fixing errors, and sending push notifications without user prompting. It's not shipped yet, but it signals the direction.
  • Claude Code's leaked memory architecture uses a three-layer system with a lightweight pointer index (memory.md) that stores locations rather than data, grepping for specifics only when needed — a pattern worth understanding for anyone building context-aware AI systems.
  • OpenAI's $122B raise (largest ever) buries the real news: they're building a 'unified AI super app' merging chat, GPTs, Codex, browsing, and agentic capabilities into one agent-first experience — consolidation over fragmentation.
  • Open-weight LLMs (Gemma 4, Qwen 3.5/3.6, Arcee Trinity) are rapidly approaching frontier model performance. Gemma 4 runs on laptop GPUs under Apache 2.0, making local agentic workflows increasingly viable.
  • Microsoft's MAI-Transcribe-1 significantly outperforms Whisper, GPT Transcribe, and Gemini Flash on word error rate across 25 languages — available now in Microsoft Foundry for developers.
  • Sora was losing roughly $1M/day before shutdown — a concrete reminder that even well-funded AI products need sustainable unit economics.
  • The broader trend across Anthropic, OpenAI, and Salesforce/Slack is toward proactive, always-on AI agents that act autonomously rather than waiting for prompts — the 'post-prompting era.'
