Building Voice Agents with Gemini 3.1 Flash Live: Free Playground to Production Demos

TL;DR

Google released Gemini 3.1 Flash Live, a speech-to-speech voice model with vision, tool calling, and low latency. This walkthrough covers using it free in Google AI Studio, customizing agents with system instructions, and building two demo integrations (a product website voice widget and a personal assistant with ClickUp/Calendar tools) using Claude Code — all built in under 30 minutes each.

What is Gemini 3.1 Flash Live and Why It Matters

Google released Gemini 3.1 Flash Live, which they're calling their biggest upgrade yet. The key architectural difference: it's speech-to-speech, not speech-to-text-to-speech. This means the model interprets audio directly rather than transcribing it first, which enables better contextual awareness — it can understand sarcasm, stress, and frustration from vocal cues rather than just words.

Google's benchmarks show it outperformed the previous Gemini 2.5 Flash model by approximately 19% on multi-step function calling. On the Audio Multi-Challenge benchmark, it outperformed both previous Gemini models and models from other providers. A major claimed improvement is handling noisy environments — traffic, horns, restaurant noise — which matters for real-world business voice agent deployments.

The model also has improved accuracy with alphanumeric strings (since it's interpreting speech directly rather than relying on transcription), and it supports over 70 languages, making it potentially useful as a real-time translator.

Getting Started: Free Access in Google AI Studio

You can try Gemini 3.1 Flash Live for free in Google AI Studio. No billing information or API key is required for the free tier.

Steps to access:

1. Go to Google AI Studio

2. On the main screen, look at the top right for the model selector

3. By default it may be set to "all" or another model — click on the "Live" tab

4. Select "Gemini 3.1 Flash Live Preview"

5. You'll see pricing information below, but notice it says "no API key" — meaning you're on the free plan

The creator noted the free tier has limitations: your data is used by Google to improve products, rate limits are much lower (roughly 3 active sessions), and you don't get features like context caching or batch API. On the paid tier, you get enterprise-grade privacy (data not used by Google), higher rate limits, and additional features.

First Impressions: Latency and Interruption Handling

The creator's first observation was the speed improvement. When testing in Google AI Studio, the model responded quickly and handled interruptions naturally. With previous voice agents, interrupting would create an awkward back-and-forth ("Oh sorry, no you go"). With Gemini 3.1 Flash Live, when interrupted, the model stops immediately and lets you speak. The creator demonstrated this by cutting the model off mid-sentence and it stopped instantly, which felt significantly more human.

The creator also demonstrated the webcam/vision capability. The model correctly identified the room details (light gray walls, shelf with trophies, microphone), identified the microphone as a Shure MV7, and identified the mic arm as a Rode arm — all from the webcam feed.

Customizing Agents with System Instructions

Google AI Studio lets you build custom voice agents directly in the interface using system instructions.

Steps:

1. In the Google AI Studio interface, find the system instructions area

2. Create instructions and save them with a name (the creator named his "my coach")

3. Write your prompt with tone, style, and behavior instructions

The creator's example prompt was something like: "You are my personal fitness coach. You will help me get on a plan to eat healthier, build muscle, stuff like that. And you also have a very strong Scottish accent."

After saving, the agent immediately adopted the persona. When asked about taking a 3-week gym break, it responded in a Scottish accent and asked relevant follow-up questions about the reason for the break. The creator noted the accent came and went, which was amusing but demonstrated how quickly you can customize agent behavior.

Studio Configuration Options

Beyond system instructions, Google AI Studio exposes several configuration options:

- **Voice selection**: Multiple voice options to choose from (must end the session to change)

- **Media resolution**: Adjustable setting for webcam/vision input

- **Thinking level**: Options for no thinking (default), low, medium, or high thinking

- **Session context**: Configurable max context and target context sizes (described as "a lot bigger")

- **Function calling**: Toggle to enable tool/function use

- **Automatic function response**: Additional function calling setting

- **Grounding with Google Search**: When enabled, the agent performs Google searches on the backend to answer questions

For function calling, you can define functions by clicking "edit" after toggling it on, but you'd need to provide actual code snippets for the tools. This is where the Gemini Live API becomes relevant — it lets you connect the voice model to external systems like websites or phone numbers rather than being limited to Google AI Studio.

Demo 1: Product Website Voice Widget (Apex Keyboards)

The first demo was a fake keyboard website called "Apex" with an embedded Gemini 3.1 Flash Live voice agent. The site was described as "very vibe coded" and built as a quick proof of concept.

The implementation had a button on the site that, when clicked, opened a voice agent widget. The agent had been given knowledge about the Apex brand and products. In the demo:

- The agent explained what Apex makes (premium mechanical keyboards with great switches, customizable features, durable builds)

- When asked for a recommendation, it asked clarifying questions (gaming, typing, or office work?)

- The creator said they'd be working in an office with some travel, and the agent adapted its recommendation

- The creator chose the 60% model and asked about shipping; the agent responded with "free standard shipping takes 5 to 7 business days" before being interrupted

The creator emphasized how conversational and natural the interaction felt, and that you can give the agent domain-specific knowledge.

Demo 2: Personal Assistant 'Arya' with Calendar and ClickUp Integration

The second demo was a personal assistant named "Arya" with actual tool integrations — specifically Google Calendar and ClickUp. The creator acknowledged the interface was ugly but wanted to demonstrate function calling capabilities.

In the live demo:

1. Asked Arya to look at tomorrow's calendar (Saturday, March 28th)

2. Arya called the calendar function and returned: a walk from 9:00 AM to 10:00 AM and a meeting from 12:30 PM to 1:30 PM

3. Asked Arya to block off 3:00 PM to 5:00 PM called "research block" — Arya created the calendar event

4. Asked Arya to look up the "research queue" in ClickUp under the "Upai" workspace, specifically in the "Upai Consulting" space

5. Arya navigated to the correct list and found one existing task called "Chipotle"

6. Asked Arya to add a new task called "Microsoft" — Arya confirmed it was added

Both demos were built as localhost applications because connecting the websocket locally is straightforward. The creator stated both sites and integrations were built in under 30 minutes each using Claude Code.

How the Demos Were Built with Claude Code

The creator's workflow for building both demos:

1. Opened a fresh Claude Code project

2. Fed it the Gemini API documentation — specifically the API docs, the Live API docs, and the function calling documentation

3. Had Claude Code analyze all the documentation

4. Optionally: asked Claude Code to create a resource guide markdown file (e.g., "Gemini flash MD") to store the information for later reference

5. Asked Claude Code questions: "How does this work? What can you do? What should we do as a demo?"

6. Claude Code helped come up with the two different demo use cases

7. Spun up two separate Claude Code sessions, one per demo, providing API keys and the use case description

The creator organized everything into a single project folder and planned to drop it into their Free School community (link in the video description) along with a resource guide explaining what the project does and where to find things.

The creator emphasized: "The best way to learn is just to get your hands dirty and to ask a ton of questions."

Gotcha: Function Calling Causes Silent Pauses

A significant current limitation: when the model calls a function/tool, it stops speaking entirely and waits for the response. There is no way to prompt it to keep talking while the function executes in the background.

The creator explained: "A lot of times what you want to do with voice agents is you want to have them keep talking as they are calling a function because they call the function and they have to wait for the response. You can usually prompt them and have them working out so that they talk so there's not awkward silences. But right now with this Gemini 3.1 Flash Live, it doesn't do that."

The creator described this as a synchronous vs. asynchronous issue and expects it to be resolved in time, but it's something to be aware of when building with this model today.

Gotcha: Deploying to Production Is Not Trivial

Both demos ran on localhost because connecting the websocket locally to Gemini is straightforward. Pushing to a live production website or phone number is significantly more complex than using a platform like ElevenLabs.

The creator contrasted the two approaches:

- **ElevenLabs**: Handles hosting, manages the persistent websocket connection, and lets you embed a widget with essentially one click

- **Gemini Live**: Requires a persistent server process. Deploying to Vercel or adding the widget to an existing live site requires thinking about key management, connection handling, and server infrastructure

The creator noted Claude Code can get you there if you give it structured planning information and the right documentation, but warned: "It's not one click of a button the same way that ElevenLabs is." You'll need to go back and forth more and think about how to transfer over keys, connections, and related infrastructure.

Pricing Breakdown

The free tier requires no API key or billing information. Once you move to paid:

- Pricing is per-token with separate charges for input tokens and output tokens

- A roughly 10-minute voice call costs approximately 14 cents

- Paid tier benefits: enterprise-grade data privacy (Google does not use your data to improve products), higher rate limits, context caching, and batch API access

- Free tier limitations: data used by Google, approximately 3 active sessions, lower rate limits

The creator's recommendation: stay on the free tier while testing and experimenting.

Key Takeaways

  • Gemini 3.1 Flash Live is free to try in Google AI Studio with no API key required — just select the 'Live' tab and choose the model
  • The speech-to-speech architecture (not speech-to-text-to-speech) means lower latency, better interruption handling, and contextual understanding of tone/emotion
  • You can build custom voice agents directly in Google AI Studio using system instructions with saved presets — no code required for basic experimentation
  • For production integrations with tools (calendar, task management, etc.), use Claude Code with the Gemini Live API documentation to build websocket-based applications
  • Current limitation: function calling is synchronous — the agent stops speaking while waiting for tool responses, creating awkward silences
  • Deploying Gemini Live voice agents to production websites requires managing persistent websocket connections and server infrastructure — significantly more complex than widget-based platforms like ElevenLabs
  • A 10-minute voice call costs roughly 14 cents on the paid tier; free tier works for testing but has lower rate limits and uses your data for Google product improvement
