Rollout curve

  • Month 1: rough — hallucinations, broken runs, model resets, 'gateway restart a bunch of times.'

Skills, harnesses, and memory

  • Siu pushes the slogan 'fat skills, thin harnesses': the harness (e.g. Claude Code as the driver) should stay thin while the `skill.md` files hold deep, detailed SOPs.
  • He recommends recording Loom videos or dictating with WhisperFlow, then turning those captures into skill files.
  • For long-term memory he uses Obsidian because agents 'love to read markdown files.' He pans through a vault containing folders for shared decisions, handoffs, knowledge, projects, and a daily log.
  • He opens a project note documenting a ManyChat integration whose job is to take an Instagram comment and route the right lead magnet to the commenter.
  • His justification: agents forget context across sessions, so a markdown-based memory store gives them durable, fast-retrieval knowledge instead of re-hallucinating.

Key takeaways

  • The architecture is straightforward: pipe every company data source into one queryable layer, then let both humans and a fleet of named, role-specific agents draw from it.
  • 'Fat skills, thin harnesses' — invest in detailed, well-documented skill.md files; the driver (e.g. Claude Code) should stay minimal.
  • Persistent markdown memory (Obsidian) is the antidote to agents forgetting context across sessions.
  • Expect a rough month 1; the value compounds in months 2–3 as patterns emerge and agents get reused.
  • Sandbox each agent (he uses NemoClaw) so one bad actor or broken agent can't take down the rest.
  • Concrete ROI example: an Alfred-run vendor/ops audit surfaced ~$500K/year in cuttable spend (CRM seats, redundant SEO tools, underperforming ads).

Gotchas & caveats

  • Month 1 is painful: hallucinations, broken runs, frequent model and gateway restarts.
  • Without solid 'plumbing' (skills, memory, data wiring) the brain won't compound — it just stays unreliable.
  • Don't grant agents permission to send emails out of the gate; treat send-authority as a separate, later decision.
  • Falling token prices won't shrink your bill — usage will scale up to fill the headroom; plan for token optimization.
  • This is a vendor-pitch video for singlebrain.com; the strongest claims (10x lifts, 100x speed, $500K savings) are self-reported and uncorroborated.
