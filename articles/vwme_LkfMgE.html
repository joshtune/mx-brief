Walkthrough

The deal

  • On Tuesday, April 21st, SpaceX posted on X that it had signed a deal with Cursor containing an unusual clause: SpaceX can either pay Cursor $10B for working together, or buy the company outright for $60B later this year.
  • Context given: Cursor was valued at $2.5B 15 months ago, then $9B, $29B, and now $60B. Customers include Nvidia, Stripe, and more than half of the Fortune 500.
  • Surrounding context: xAI merged into SpaceX in February; OpenAI was an early Cursor investor; the Musk vs. Altman trial starts the following week.

What Cursor actually is

  • The example: telling Cursor 'add OAuth to login flow' results in edits across ~8 files — controller, middleware, tests — surfaced as a clean diff for one-click apply.
  • Cursor uses the same models you can already access (GPT, Claude, Gemini), so the differentiator isn't the model — it's how the right context is selected.

The core problem

  • Production codebases (~50,000 files) won't fit in any context window. The hard problem is picking the right files to show the model, every time. The video frames this as 'the entire game'.

Indexing pipeline

  • On project open, Cursor reads every file locally and filters out noise: dotfolders, node_modules, build outputs.
  • It chunks files using Tree-sitter, a structural parser, so chunks are whole functions, classes, or logical blocks — not arbitrary line slices.
  • It builds a Merkle tree: each file gets a hash, each folder gets a hash derived from its children, up to the root. When one file changes, only the fingerprints along that path change, so Cursor knows exactly what to re-index instead of reprocessing the whole repo.

Embedding and storage

  • Each chunk is converted into a vector capturing meaning, not text. The example: searching 'login' would miss `authenticate.ts`; vector search puts auth code in the same neighborhood regardless of vocabulary.
  • Vectors are stored in Turbopuffer, Cursor's vector database — described as 'Google for your codebase, but search by meaning'.
  • Privacy: raw source never leaves the machine. Only vectors are sent to the server, file names are obfuscated, and chunks are encrypted.

Retrieval at query time

  • The user prompt (e.g. 'refactor the login flow to support Google auth') is embedded into the same vector space.
  • A nearest-neighbor search returns the top-matching chunks.
  • Critically, Cursor then 'follows the code': starting from a top match like the OAuth controller, it walks imports, callers, and callees to pull in the surrounding web of related code — mimicking how a senior engineer traces flow.
  • It assembles a structured prompt: question on top, relevant code, project rules — and sends that focused brief (not the whole repo) to the model.

Execution loop

  • Cursor generates a diff, shows the proposed changes, and applies them on click.
  • If something breaks, Cursor reads the error and retries.
  • Cursor 2.0 introduced Composer, an in-house model trained specifically for tool use (search, edit, run) via reinforcement learning inside real codebases. The presenter attributes most tasks finishing in under 30 seconds to this loop plus fast retrieval.

Why the deal makes sense

  • Reason 1 — xAI is losing the coding war. OpenAI has Codex, Anthropic has Claude (described as dominating agentic coding); Grok isn't a serious choice for production code. Buying Cursor instantly puts xAI in the agentic-coding market with existing Fortune 500 customers.
  • Reason 2 — Own the interface layer. Three-layer stack: infra (SpaceX already runs the Colossus cluster, ~1M H100-equivalent chips), models (commoditizing as more frontier models compete on price), and interface (Cursor, Copilot, Windsurf). The interface is 'the remote control' — whoever owns it owns distribution, workflow, and developer-behavior data.
  • Reason 3 — IPO narrative. A pure space company gets a space-company multiple; a vertically integrated AI platform on the world's largest GPU cluster with the fastest-growing dev tool on top gets a different multiple. Cursor is described as gross-margin positive with a $6B revenue run rate.

Key takeaways

  • The model isn't the moat — context selection is. Cursor's edge is choosing which slice of a 50k-file repo to send to a generic frontier model.
  • Tree-sitter chunking + Merkle-tree hashing is what makes incremental indexing tractable on large repos; only changed paths get re-embedded.
  • Retrieval isn't just nearest-neighbor — Cursor expands matches by walking imports and callers, which is what makes results feel 'senior-engineer-like'.
  • Privacy model: code stays local, only vectors leave; file names are obfuscated and chunks encrypted.
  • The strategic thesis: as frontier models commoditize, the interface layer (where the developer lives daily) becomes the most valuable layer in the AI stack.
  • The Cursor deal also serves SpaceX's pre-IPO narrative shift from 'rocket company' to 'vertically integrated AI platform'.

Gotchas & caveats

  • Plain text search misses semantically equivalent code (the `login` vs `authenticate.ts` example) — keyword-only retrieval is insufficient for an agent.
  • You can't just dump a repo into a model: context windows choke on real production codebases, which is why selection matters more than raw model capability.
  • The video is conceptual, not hands-on — there are no commands, configs, or rules-file examples; the promised 'Cursor deep dive' on tips/tricks is gated behind viewer comments.
